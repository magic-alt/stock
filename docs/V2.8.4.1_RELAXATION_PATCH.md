# V2.8.4.1 - 策略放宽补丁说明

## 问题诊断

V2.8.4 首次发布的增强策略出现"全年0笔交易、指标NaN"的两个原因：

### 1. 触发条件过严

**MACD_RegimePullback (`macd_r`)**:
- ❌ **AND逻辑**：`EMA200斜率>0` **AND** `ROC100>0` → 2023-2024年茅台几乎不满足
- ❌ **Warmup过长**：`max(200, 100, 20, 14, 38) = 200根` → 484根数据中前41%不交易
- ❌ **回落条件单一**：仅允许 `low <= ema20-k*ATR` 且 `close > ema20`
- ❌ **ATR未初始化**：初期ATR=0导致止损/TP逻辑失效

**Bollinger_EnhancedStrategy (`boll_e`)**:
- ❌ **反弹检测严格**：仅检查 `close[-1] < bot[-1]` AND `close[0] > bot[0]`（昨日必须低于下轨）
- ❌ **固定warmup**：`warmup_bars=30` 硬编码
- ❌ **长期持仓风险**：无超时退出机制，可能"拿到死"

### 2. 边界情况处理不足

- ❌ **索引越界**：`close[-1]`、`bot[-1]` 在初期可能越界
- ❌ **ATR=0/NaN**：初期ATR未稳定导致止损计算错误
- ❌ **状态变量未重置**：`notify_trade` 可能遗漏状态清理

## V2.8.4.1 修复方案

### A. MACD_RegimePullback 放宽 (9项优化)

#### 1. 趋势逻辑放宽
```python
# 新增参数
("trend_logic", "or"),  # 'or' | 'and'

# 实现
if self.params.trend_logic.lower() == "and":
    regime_ok = ema_up and roc_up
else:
    regime_ok = ema_up or roc_up  # 默认：EMA OR ROC
```

#### 2. 动态 Warmup
```python
self.warmup_bars = max(
    self.params.ema_trend_period,  # 200
    self.params.roc_period,        # 100
    self.params.ema_entry_period,  # 20
    self.params.atr_period,        # 14
    self.params.slow + self.params.signal + 2  # 38
)
# 结果：warmup=200根（保持不变但显式计算）
```

#### 3. 入场条件双路径
```python
# 条件A：经典回落-反弹
cond_a = (low <= pullback_line) and (close > ema20)

# 条件B：温和版（新增）
cond_b = (close <= ema20) and (macd.macd[0] > macd.macd[-1])

# 满足任一即可
if cond_a or cond_b:
    self.order = self.buy()
```

#### 4. ATR 降级保护
```python
def _atr_safe(self) -> float:
    try:
        import math
        v = float(self.atr[0])
        if not math.isfinite(v) or v <= 0:
            return 0.0
        return v
    except Exception:
        return 0.0

# 使用时：
atr = self._atr_safe()
r = max(self.params.atr_sl_mult * atr, 0.01 * close)  # 最小用1%
```

#### 5. 参数默认值放宽


#### 6. 状态重置完善
```python

        self.last_exit_bar = len(self)
        self.entry_bar = None
        self.entry_price = None
```

#### 7-9. 其他细节

- ✅ 完整的 `_after_exit()` 状态清理

### B. Bollinger_Enhanced 放宽 (8项优化)

#### 1. 反弹检测回溯窗口
```python
# 新增参数
("rebound_lookback", 3),  # 最近N根

# 实现
lookback = min(int(self.params.rebound_lookback), i - 1)
below_recent = any(
    float(self.data.close[-k]) < float(self.bot[-k]) 
    for k in range(1, lookback + 1)
)
rebound_now = close > float(self.bot[0])

if trend_ok and below_recent and rebound_now:
    self.order = self.buy()
```
**改进**：不再要求"昨日必须低于下轨"，而是"最近3根任一根低于下轨即可"。

#### 2. 超时退出机制
```python
# 新增参数
("max_hold", 60),  # 最长持有60根

# 实现
if (i - (self.entry_bar or i)) >= int(self.params.max_hold):
    if close >= float(self.mid[0]):  # 回到中轨附近
        self.order = self.close()
        self._mark_exit(i)
        self.log(f"TIME EXIT @ {close:.2f}")
        return
```

#### 3. 动态 Warmup
```python
self._warmup = (
    self.params.warmup_bars
    if self.params.warmup_bars is not None
    else max(self.params.period, self.params.atr_period, 30)
)
# 结果：warmup=max(20, 14, 30)=30根（自动计算）
```

#### 4. ATR 降级保护
```python
def _atr_safe(self) -> float:
```

**方案A**: 条件式warmup

```python
total_bars = len(self.data)

                            self.params.atr_period + 5,
                            self.params.slow + self.params.signal + 2)
else:
    # 长期数据：使用完整warmup
    self.warmup_bars = max(self.params.ema_trend_period,
                            self.params.roc_period, ...)
```

```python
if total_bars >= 400:
```
