# GUI 错误修复说明

## 问题总结

GUI 运行时出现以下错误：
1. 基准指数 `000300.SS` 加载失败
2. 策略收到不支持的 `period` 参数
3. EMA 策略数据为空

## 已修复内容

### 1. 基准指数默认值修改
- **修改位置**: `backtest_gui.py` 第 185 行
- **修改内容**: 将默认基准从 `000300.SS` 改为 `000300.SH`
- **原因**: AKShare 更好地支持 `.SH` 后缀的指数代码

### 2. 策略参数输入框清空
- **修改位置**: `backtest_gui.py` 第 275、478 行
- **修改内容**: 移除了默认的 `{"period": 20}` 参数示例
- **原因**: 不同策略有不同的参数要求，空白让用户使用策略默认参数更安全

### 3. 增强 _standardize_index_frame 函数
- **修改位置**: `src/data_sources/providers.py` 第 124-168 行
- **修改内容**: 更健壮的列名处理和日期索引转换
- **原因**: AKShare 返回的指数数据格式可能变化

### 4. 添加 JSON 参数错误处理
- **修改位置**: `backtest_gui.py` _run_single_backtest 方法
- **修改内容**: 添加 try-except 捕获 JSON 解析错误
- **原因**: 防止格式错误的 JSON 导致程序崩溃

### 5. 增强数据预览功能
- **修改位置**: `backtest_gui.py` preview_data 方法
- **修改内容**: 添加更详细的日志输出和错误处理
- **原因**: 帮助用户快速定位数据加载问题

## 使用建议

### 正确的使用方式

1. **基准指数**
   - 使用 `000300.SH` 而不是 `000300.SS`
   - 或留空不使用基准

2. **策略参数**
   - 留空（推荐）- 使用策略默认参数
   - 或查看策略详情后填写正确的参数格式

3. **日期范围**
   - 建议使用至少 6 个月的数据
   - 短期数据（如 3 个月）可能不足以计算某些指标

### GUI 使用步骤

#### 快速开始（3步）

```
1. 数据配置标签页:
   - 股票代码: 600519.SH (每行一个)
   - 日期: 2023-01-01 ~ 2024-12-31
   - 基准: 000300.SH (或留空)
   - 点击"预览数据"验证

2. 策略配置标签页:
   - 选择策略: ema (点击选中)
   - 参数: 留空（使用默认）

3. 优化配置标签页:
   - 运行模式: 单次回测
   - 点击"开始回测"
```

#### 验证步骤

```
1. 点击"预览数据"按钮
2. 查看右侧日志:
   - ✓ 成功加载 1 只股票的数据
   - 600519.SH: XX 条记录
   - ✓ 基准指数加载成功
3. 如果显示错误，根据提示调整配置
4. 确认无误后点击"开始回测"
```

## 常见错误及解决方法

### 错误 1: 基准指数加载失败
```
Failed to load index 000300.SS: 'date'
```
**解决**: 将基准改为 `000300.SH` 或留空

### 错误 2: 策略参数不匹配
```
ADXTrendStrategy.__init__() got an unexpected keyword argument 'period'
```
**解决**: 清空策略参数输入框，使用默认参数

### 错误 3: 数据为空
```
EMA period (20) requires at least 20 bars of data, but only 0 bars available
```
**解决**: 
- 检查股票代码格式（需要 .SH 或 .SZ 后缀）
- 延长日期范围（至少 6 个月）
- 点击"预览数据"验证数据是否成功加载

### 错误 4: JSON 格式错误
```
策略参数 JSON 格式错误
```
**解决**: 
- 检查 JSON 语法（使用在线 JSON 验证器）
- 或直接清空参数输入框
