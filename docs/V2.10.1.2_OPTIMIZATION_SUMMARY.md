### 2. ✅ adj_type 说明文档

**新增文件**: `docs/ADJ_TYPE_EXPLANATION.md`

#### 内容概要
- **复权概念**: 什么是复权，为什么需要复权
- **三种类型详解**:
  - `noadj`: 不复权（原始价格）
  - `qfq`: 前复权（保持最新价不变，技术分析推荐）
  - `hfq`: 后复权（保持历史价不变，长期收益分析）
- **使用场景**: 各种复权类型的最佳应用场景
- **CLI 示例**: 如何在命令行中指定复权类型
- **Python API**: 如何在代码中使用不同复权类型
- **常见问题**: 10个常见问题解答

#### 推荐使用
- **技术分析回测**: `qfq` (前复权) ✅
- **基本面分析**: `hfq` (后复权)
- **实盘对照**: `noadj` (不复权)

#### 代码注释增强
在 `_update_metadata()` 方法中增加了详细注释：
```python
adj_type: Adjustment type (noadj/qfq/hfq)
    - noadj: 不复权 (原始价格)
    - qfq: 前复权 (向前复权，保持最新价格不变)
    - hfq: 后复权 (向后复权，保持历史价格不变)
```

### 3. ✅ --plot 自动保存报告

**修改文件**: 
- `src/backtest/plotting.py`
- `unified_backtest_framework.py`

#### 新增功能参数
```python
def plot_backtest_with_indicators(
    cerebro: bt.Cerebro,
    ...
    auto_save: bool = False,           # 新增：自动保存到report目录
    strategy_name: str = "strategy",   # 新增：策略名称
    symbols: List[str] = None,         # 新增：股票代码列表
) -> Optional[str]:                    # 返回报告目录路径
```

#### 目录命名规则
```
report/
└── {股票代码}_{策略名}_{时间戳}/
    ├── backtest_result.png      # PNG格式图表
    └── backtest_result.pkl      # 原生matplotlib格式
```

**示例**:
```
report/
└── 601318_macd_20251026_214028/
    ├── backtest_result.png
    └── backtest_result.pkl
```

#### 自动保存流程
1. 检测到 `--plot` 选项
2. 自动创建 `report` 目录
3. 按 `{股票}_{策略}_{时间}` 创建子目录
4. 保存 PNG 和 PKL 两种格式
5. 输出报告路径

#### CLI 使用
```bash
# 使用--plot选项，自动保存报告
python unified_backtest_framework.py run \\
    --strategy macd \\
    --symbols 601318.SH \\
    --start 2020-01-01 \\
    --end 2024-12-31 \\
    --plot              # 自动保存到report/
```

### 4. ✅ 保存原生 matplotlib 格式

**新增功能**: 同时保存 PNG 和 pickle 格式

#### 保存格式

**PNG 格式** (`backtest_result.png`):
- 高清图片（300 DPI）
- 适合查看和分享
- 不可编辑

**Pickle 格式** (`backtest_result.pkl`):
- 原生 matplotlib Figure 对象
- 可重新编辑和调整
- Python pickle 序列化

#### 加载 pickle 图表
```python
import pickle
import matplotlib.pyplot as plt

# 加载保存的图表
with open('report/601318_macd_20251026_214028/backtest_result.pkl', 'rb') as f:
    fig = pickle.load(f)

# 可以继续编辑
fig.suptitle('修改后的标题', fontsize=16)

# 重新保存
fig.savefig('modified_chart.png', dpi=300)

# 或显示
plt.show()
```

#### 优势
- **可编辑性**: 保留完整的 matplotlib 对象
- **灵活性**: 可以重新调整样式、添加注释
- **分析性**: 可以提取数据进行进一步分析

## 数据库迁移说明

### 对于新数据库
- 无需迁移，新保存的数据自动包含 `name` 字段

### 对于现有数据库

**方法1: 重新导入数据（推荐）**
```bash
# 删除旧数据库
rm cache/market_data.db

# 重新导入CSV
python -c "
from src.data_sources.db_manager import SQLiteDataManager
db = SQLiteDataManager('./cache/market_data.db')
stats = db.batch_import_from_cache('./cache')
print(f'导入: {stats[\"success\"]} 个文件')
"
```

**方法2: 手动添加 name 列**
```sql
-- 添加 name 列
ALTER TABLE metadata ADD COLUMN name TEXT;

-- 更新现有数据（需要手动填充）
UPDATE metadata SET name = '贵州茅台' WHERE symbol = '600519.SH';
UPDATE metadata SET name = '中国平安' WHERE symbol = '601318.SH';
-- ... 继续更新其他股票
```

**方法3: Python 脚本批量更新**
```python
import sqlite3
from src.data_sources.db_manager import SQLiteDataManager

db = SQLiteDataManager('./cache/market_data.db')

# 连接数据库
conn = sqlite3.connect('./cache/market_data.db')
cursor = conn.cursor()

# 添加列（如果不存在）
try:
    cursor.execute("ALTER TABLE metadata ADD COLUMN name TEXT")
except:
    pass  # 列已存在

# 获取所有symbol
cursor.execute("SELECT DISTINCT symbol FROM metadata")
symbols = [row[0] for row in cursor.fetchall()]

# 批量更新
for symbol in symbols:
    name = db._get_symbol_name(symbol)
    cursor.execute("UPDATE metadata SET name = ? WHERE symbol = ?", (name, symbol))

conn.commit()
conn.close()

print(f"✓ 已更新 {len(symbols)} 个标的的名称")
```
