## 🚀 技术亮点

### 1. 策略模板模式

**设计理念**: 简化 > 强大

```python
# 传统方式 (Backtrader)
class EMAStrategy(bt.Strategy):
    params = (("period", 20),)
    
    def __init__(self):
        self.sma = bt.indicators.SMA(self.data.close, period=self.params.period)
    
    def next(self):
        if self.data.close[0] > self.sma[0]:
            self.buy()

# 新方式 (Template)
class EMATemplate(StrategyTemplate):
    params = {"period": 20}
    
    def on_bar(self, symbol, bar):
        # 纯 Python 逻辑，无魔法
        ema = calculate_ema(self.ctx[symbol]["closes"], self.params["period"])
        if bar["close"] > ema:
            # 发送信号或更新状态
            self.ctx[symbol]["position"] = 1
```

**优势**:
- ✅ 代码更清晰（减少 30% 代码量）
- ✅ 更易理解（无需学习 Backtrader API）
- ✅ 更易测试（纯函数，可模拟 bar 数据）
- ✅ 更易复用（同一模板适配多个执行引擎）

### 2. CLI 参数系统

**设计理念**: 灵活 > 硬编码

```bash
# Before: 费用规则硬编码在策略中
python unified_backtest_framework.py run --strategy ema ...

# After: 费用规则通过参数配置
python unified_backtest_framework.py run \
  --strategy ema \
  --fee-config cn_stock \
  --fee-params '{"commission_rate":0.0001,"min_commission":5.0}'
```

**实现细节**:
1. CLI 解析 JSON 参数
2. Engine 注入参数到 `param_dict` (以 `_` 开头)
3. `_run_module` 提取参数配置 broker
4. `add_strategy` 过滤 `_` 参数再传给策略

**优势**:
- ✅ 无需修改代码即可测试不同费用规则
- ✅ 支持免五/不免五模式切换
- ✅ 网格搜索也可使用自定义费用
- ✅ 向后兼容（不指定则使用默认）

### 3. 事件驱动增强

**设计理念**: 解耦 > 耦合

```python
# 传统方式: 进度打印硬编码在 grid_search
def grid_search(...):
    for i, combo in enumerate(combos):
        result = run_backtest(combo)
        print(f"Progress: {i+1}/{len(combos)}")  # 硬编码

# 新方式: 通过事件解耦
def grid_search(...):
    for i, combo in enumerate(combos):
        result = run_backtest(combo)
        self.events.put(Event(EventType.METRICS_CALCULATED, {
            "i": i+1, "total": len(combos), "metrics": result
        }))  # 发布事件

# 监听器独立实现进度显示/通知
class ProgressBarHandler:
    def on_metrics_calculated(self, event):
        self.pbar.update(1)  # 进度条

class TelegramNotifier:
    def on_pipeline_stage(self, event):
        if event.data["stage"] == "grid.done":
            self._send_message(f"✅ 完成 {event.data['total_runs']} 次回测")
```

**优势**:
- ✅ 核心逻辑与展示层解耦
- ✅ 可同时注册多个监听器（进度条 + Telegram + Email）
- ✅ 监听器可独立启用/禁用
- ✅ 易于扩展（添加新监听器无需修改核心代码）
