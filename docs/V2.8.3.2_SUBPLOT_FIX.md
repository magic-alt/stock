## 根本原因分析

### 问题来源

在 V2.8.3 中，我们为了丰富技术指标，添加了多个子图：

```python
# 问题代码 (V2.8.3)
bt.indicators.WeightedMovingAverage(data, period=25, subplot=True)     # 创建子图3
bt.indicators.RateOfChange(data, period=12, subplot=True)              # 创建子图4  
bt.indicators.Momentum(data, period=14, subplot=True)                  # 创建子图5
```

### 为什么会模糊不清？

**WMA (加权移动平均)** 的值与价格范围相同（如 1400-1700），但显示在独立子图上：
- 主图（子图1）: 完整的价格 + K线 + 其他指标
- WMA子图（子图3）: 只有一条 WMA 线，没有K线背景

因此看起来像"模糊的价格副本"——实际上是一条孤零零的移动平均线。

### 为什么会空白？

**ROC (变动率指标)** 和 **Momentum (动量指标)** 的值通常很小或接近0：
- ROC: 计算价格变动百分比，通常在 -5% 到 +5% 之间
- Momentum: 计算价格差值，可能在 -50 到 +50 之间

如果数据不活跃或变动很小，这些子图可能看起来是空白的（实际上有数据，但太小看不见）。

### 子图布局问题

**V2.8.3 的布局**:
```
图表总共有 10+ 个子图：
1. 价格图 (主图)
2. 成交量
3. WMA (模糊不清) ❌
4. ROC (空白) ❌
5. Momentum (空白) ❌
6. MACD
7. ADX
8. RSI
9. Stochastic
10. CCI
...
```

**问题**:
- 子图太多，每个子图高度被压缩
- WMA 没有K线背景，看起来很怪异
- ROC/Momentum 数值太小，看起来是空白
- 图表整体阅读体验差

## 解决方案

### 核心原则

**保留所有指标的计算逻辑，但优化显示方式**：
1. **主图指标**: 与价格相关的指标显示在主图上（均线、布林带）
2. **有意义的子图**: 只为真正需要独立显示的指标创建子图（MACD、RSI等）
3. **隐藏显示**: 对于内部计算用的指标，设置 `plot=False`

### 具体修改

#### 1. WMA 移到主图

```python
# 修改前
bt.indicators.WeightedMovingAverage(data, period=25, subplot=True)

# 修改后
bt.indicators.WeightedMovingAverage(data, period=25, plotname='WMA_25')
# 移除 subplot=True，默认显示在主图上
```

**效果**: WMA 与 SMA、EMA 一起显示在价格图上，不再创建模糊的子图。

#### 2. ROC 和 Momentum 隐藏显示

```python
# 修改前
bt.indicators.RateOfChange(data, period=12, subplot=True)
bt.indicators.Momentum(data, period=14, subplot=True)

# 修改后
bt.indicators.RateOfChange(data, period=12, plot=False)
bt.indicators.Momentum(data, period=14, plot=False)
```

**效果**: 
- 指标仍然被计算（策略可以使用）
- 不创建空白子图
- 图表更简洁

#### 3. Volume SMA 优化

```python
# 确保 Volume SMA 显示在成交量子图上
if hasattr(data, 'volume'):
    bt.indicators.SMA(data.volume, period=20, subplot=True, plotname='Volume_SMA')
```

**效果**: Volume SMA 与成交量柱状图一起显示，便于对比。

## 常见问题

### Q1: 为什么 WMA 要移到主图？

**A**: WMA (加权移动平均) 本质上是一条均线，与价格范围相同。将它与其他均线（SMA、EMA）一起显示在主图上，便于对比不同均线的走势。创建独立子图反而会让它看起来"模糊不清"。

### Q2: ROC 和 Momentum 还有用吗？

**A**: 有用！这些指标仍然被计算，策略代码可以访问它们的值。只是不在图表上显示，因为：
1. 它们的数值通常很小，显示效果不好
2. 已经有 MACD、ADX 等更直观的趋势指标
3. 减少子图数量，提高图表可读性

### Q3: 能否自定义显示哪些指标？

**A**: 可以！修改 `src/backtest/plotting.py` 中的指标配置：
- 添加 `plot=False` 隐藏指标
- 移除 `subplot=True` 让指标显示在主图
- 添加 `subplot=True` 创建独立子图

### Q4: 如何添加新的指标到主图？

**A**: 
```python
# 例如：添加 DEMA (双指数移动平均) 到主图
bt.indicators.DoubleExponentialMovingAverage(data, period=30, plotname='DEMA_30')
# 不设置 subplot=True，默认显示在主图
```

### Q5: 如何显示 ROC 子图？

**A**: 如果需要查看 ROC，可以恢复子图显示：
```python
# 修改 plotting.py
bt.indicators.RateOfChange(data, period=12, subplot=True, plotname='ROC_12')
# 移除 plot=False，添加 subplot=True
```

## 总结

### 问题本质

**子图配置不当**: WMA/ROC/Momentum 创建了不必要的子图，导致布局混乱。

### 解决核心

**优化子图分配**: 
- 均线显示在主图
- 内部计算指标隐藏显示
- 只为有意义的指标创建子图

### 修复效果

- ✅ 无模糊子图（WMA 移到主图）
- ✅ 无空白子图（ROC/Momentum 隐藏）
- ✅ 布局清晰（7个有意义的子图）
- ✅ 指标完整（所有指标仍然计算）
