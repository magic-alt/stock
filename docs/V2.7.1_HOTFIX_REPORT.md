## 🎯 问题回顾

用户运行 auto pipeline 后，CSV 输出文件存在严重问题：

### 问题现象

```csv
# opt_ema.csv 中的问题行
ema,60,,,,,,,,,,,,,,,,,,,,,array assignment index out of range
ema,65,,,,,,,,,,,,,,,,,,,,,array assignment index out of range
...
ema,120,,,,,,,,,,,,,,,,,,,,,array assignment index out of range
```

**影响**:
- 13/24 行（54%）完全空白
- 只有 error 列有值，其他 22 个指标列都是空
- 用户无法分析结果，不知道是参数问题还是代码 bug

---

## 🔍 根本原因分析

### 1. 数据长度不足

**测试条件**:
- 日期范围: 2024-01-01 到 2024-03-31（3个月）
- 实际数据: 约 60 个交易日
- EMA 默认网格: [5, 10, 15, ..., 55, 60, 65, ..., 120]

**问题**:
- period ≥ 60 时，Backtrader 需要至少 60 天数据才能计算第一个有效 EMA 值
- 数据不足时，内部指标计算抛出 `IndexError: array assignment index out of range`
- 整个回测失败，没有被优雅处理

### 2. 错误处理不完善

**原始代码流程**:
```python
def _run_module(...):
    cerebro = bt.Cerebro()
    # ... 配置
    results = cerebro.run()  # ← 这里抛异常，没有 try-except
    strat = results[0]
    # ... 计算指标
```

**grid_search 中的捕获**:
```python
try:
    _, metrics, _ = self._run_module(...)
except Exception as err:
    metrics = {
        "cum_return": float("nan"),
        "final_value": float("nan"),
        "sharpe": float("nan"),
        "mdd": float("nan"),
        # ... 只有 8 个字段！缺少 15 个！
        "error": str(err),
    }
```

**问题**:
- 只返回 8 个指标，DataFrame 应该有 23 列
- 缺失的列：ann_return, ann_vol, trades, win_rate, profit_factor, etc.
- 导致 CSV 中这些列变成空白

## ✅ 修复方案

### 修复 1: 完整错误处理

**文件**: `src/backtest/engine.py`  
**方法**: `_run_module`

**变更**:

```python
@staticmethod
def _run_module(...):
    # 包裹整个 cerebro 操作
    try:
        cerebro = bt.Cerebro(stdstats=False)
        # ... 所有初始化和运行
        results = cerebro.run(runonce=True, preload=True)
        strat = results[0]
    except Exception as e:
        # 返回完整的 23 个指标字段
        warnings.warn(f"Backtest failed for params {params}: {str(e)}")
        flat_nav = pd.Series([1.0], index=[pd.Timestamp.now()], name="strategy")
        error_metrics = {
            "cum_return": float("nan"),
            "final_value": cash,
            "sharpe": float("nan"),
            "ann_return": float("nan"),
            "ann_vol": float("nan"),
            "mdd": float("nan"),
            "trades": 0,  # ← 注意：0 而不是 NaN
            "win_rate": float("nan"),
            "profit_factor": float("nan"),
            "avg_hold_bars": float("nan"),
            "avg_win": float("nan"),
            "avg_loss": float("nan"),
            "payoff_ratio": float("nan"),
            "expectancy": float("nan"),
            "exposure_ratio": float("nan"),
            "trade_freq": float("nan"),
            "calmar": float("nan"),
            "bench_return": float("nan"),
            "bench_mdd": float("nan"),
            "excess_return": float("nan"),
            "error": str(e),
        }
        return flat_nav, error_metrics, None
    
    # NAV 计算也增加保护
    try:
        timeret = pd.Series(strat.analyzers.timeret.get_analysis())
        nav = (1 + timeret.fillna(0)).cumprod()
        # ...
    except Exception as e:
        warnings.warn(f"Failed to calculate NAV: {str(e)}")
        nav = pd.Series([1.0], index=[pd.Timestamp.now()], name="strategy")
```

class EMAStrategy(bt.Strategy):
    def __init__(self):
        # 检查数据长度
        data_len = len(self.data)
        if data_len < self.params.period:
            raise ValueError(
                f"EMA period ({self.params.period}) requires at least "
                f"{self.params.period} bars of data, but only {data_len} "
                f"bars available. Please use a shorter period or longer date range."
            )
        
        # 继续正常初始化
        self.ema = bt.indicators.ExponentialMovingAverage(
            self.data.close, period=self.params.period
        )

## 💡 用户建议

### 1. 使用合理的日期范围

**推荐**:
- 至少 6-12 个月数据
- 对于大周期参数（period > 60），建议 1-2 年数据

**示例**:
```bash
# 推荐：1年数据
python unified_backtest_framework.py auto \
  --start 2023-01-01 --end 2024-12-31 \
  --symbols 600519.SH 000333.SZ \
  --strategies ema macd

# 不推荐：3个月数据（period > 40 会失败）
python unified_backtest_framework.py auto \
  --start 2024-01-01 --end 2024-03-31 \
  ...
```

### 2. 使用 hot_only 模式

**用途**: 避免不合理的参数组合

```bash
python unified_backtest_framework.py auto \
  --hot_only \  # ← 使用优化过的参数网格
  --symbols 600519.SH \
  --strategies ema macd
```

**效果**:
- EMA: period = [5, 10, 15, 20, 25, 30, 35, 40] （去掉了 45-120）
- MACD: fast/slow 范围收窄，避免无效组合
