# 复权类型说明 (Adjustment Type)

## 什么是复权？

复权是对股票价格进行调整，以消除分红、配股、送股等行为对股价造成的影响，使价格数据更具连续性和可比性。

## 三种复权类型

### 1. noadj - 不复权
- **原始价格**，未经任何调整
- 适用场景：
  - 查看真实成交价格
  - 短期交易分析
  - 实时行情显示

**示例**：
```
日期       开盘   收盘
2024-01-02  100    105
2024-01-03  (分红10元/股)
2024-01-04   90     95  (实际下跌，但因分红导致价格下调)
```

### 2. qfq - 前复权
- **向前复权**，保持最新价格不变，调整历史价格
- 适用场景：
  - **技术分析**（最常用）
  - 回测策略
  - 查看真实涨跌幅

**原理**：以最新价格为基准，向前调整历史数据

**示例**：
```
日期       不复权  前复权
2024-01-02  100    110  (调整后)
2024-01-03  (分红10元/股)
2024-01-04   95     95  (保持不变)
```

**优势**：
- ✅ 最新价格就是实际交易价
- ✅ K线图形连续，无跳空
- ✅ 技术指标计算准确

### 3. hfq - 后复权
- **向后复权**，保持历史价格不变，调整最新价格
- 适用场景：
  - 长期持有分析
  - 计算累计收益
  - 基本面研究

**原理**：以上市首日价格为基准，向后调整所有数据

**示例**：
```
日期       不复权  后复权
2024-01-02  100    100  (保持不变)
2024-01-03  (分红10元/股)
2024-01-04   95    105  (调整后，体现真实增值)
```

**优势**：
- ✅ 能看到股票从上市以来的真实增值
- ✅ 长期收益计算准确

## 回测建议

### 推荐使用 qfq (前复权)

原因：
1. **技术分析准确**：K线形态、均线、MACD等指标不会因分红产生误判
2. **价格连续性**：避免因分红造成的价格跳空
3. **策略回测真实**：信号触发时的价格接近实际成交价

### 数据库存储

系统支持同时存储三种复权类型：
```sql
-- 元数据表
CREATE TABLE metadata (
    symbol TEXT,
    adj_type TEXT,  -- 'noadj', 'qfq', 'hfq'
    ...
)

-- 同一股票可以有多个复权版本
600519.SH, noadj  -- 不复权数据
600519.SH, qfq    -- 前复权数据
600519.SH, hfq    -- 后复权数据
```

## 常见问题

### Q1: 为什么技术分析要用前复权？
**A**: 因为技术分析基于价格走势，前复权能保持K线形态连续性，避免因分红产生的虚假跳空。

### Q2: 不复权数据有什么用？
**A**: 查看真实历史成交价，对照实盘交易记录，或进行短期分析。

### Q3: 后复权数据什么时候用？
**A**: 计算长期持有收益，或进行基本面研究时使用。

### Q4: 可以混用不同复权类型吗？
**A**: 不建议。同一个策略应该使用同一种复权类型，否则可能产生错误的交易信号。

### Q5: 如何选择复权类型？
**A**: 
- 技术分析回测：使用 **qfq (前复权)** ✅
- 基本面分析：使用 **hfq (后复权)**
- 实盘对照：使用 **noadj (不复权)**
