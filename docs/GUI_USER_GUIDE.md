## 📖 简介

回测分析 GUI 是一个功能完整的图形界面程序，将命令行工具的所有功能集成到易用的可视化界面中。

**版本**: V2.7.1  
**更新日期**: 2025-10-24  
**状态**: ✅ 生产就绪

## 🚀 快速启动

### Windows 用户

双击 `启动GUI.bat` 即可启动程序。

### Linux/Mac 用户

```bash
python backtest_gui.py
```

## 📊 数据配置标签页

### 1. 数据源选择

- **akshare**: 中国 A 股（默认，无需配置）
- **yfinance**: 全球市场
- **tushare**: 中国 A 股高级接口（需要 Token）

### 2. 股票代码输入

支持多种方式输入：

**方式1: 手动输入**
```
600519.SH
000333.SZ
600036.SH
```
每行一个股票代码

**方式2: 快速选择**
- 点击"白酒股"按钮：加载白酒行业股票
- 点击"银行股"按钮：加载银行行业股票
- 点击"科技股"按钮：加载科技行业股票
- 点击"清空"按钮：清空列表

### 3. 日期范围

- **开始日期**: 格式 YYYY-MM-DD（如 2022-01-01）
- **结束日期**: 格式 YYYY-MM-DD（如 2024-12-31）
- 默认: 最近2年数据

### 4. 基准指数

- 常用指数代码:
  - `000300.SH` - 沪深300（推荐，AKShare 支持）
  - `000001.SH` - 上证指数
  - `399001.SZ` - 深证成指
  - `^GSPC` - 标普500（YFinance）
- 留空则不使用基准对比
- ⚠️ 注意：使用 AKShare 时建议用 `.SH` 后缀而不是 `.SS`

### 5. 数据预览

点击"📋 预览数据"按钮可以：
- 验证股票代码是否正确
- 查看数据是否成功下载
- 确认数据记录数量

### 6. 缓存目录

- 默认: `./cache`
- 数据会自动缓存，避免重复下载
- 点击"浏览..."可选择其他目录

## 🎯 策略配置标签页

### 1. 策略选择

**选择方式:**
- 鼠标点击: 选中单个策略
- Ctrl + 点击: 多选策略
- 全选按钮: 选择所有策略
- 清空按钮: 取消所有选择

**快速选择:**
- **趋势策略**: 适合单边行情（ema, macd, adx_trend, triple_ma, donchian）
- **震荡策略**: 适合横盘行情（bollinger, rsi, keltner, zscore）

### 2. 内置策略列表

| 策略名称 | 描述 | 适用场景 |
|---------|------|---------|
| **ema** | EMA均线交叉 | 趋势跟踪 |
| **macd** | MACD信号线 | 趋势确认 |
| **bollinger** | 布林带突破 | 震荡市场 |
| **rsi** | RSI超买超卖 | 超买超卖 |
| **adx_trend** | ADX趋势强度 | 强趋势市场 |
| **triple_ma** | 三重均线 | 多重确认 |
| **donchian** | 唐奇安通道 | 突破交易 |
| **keltner** | Keltner通道 | 趋势+波动 |
| **zscore** | Z-Score均值回归 | 均值回归 |

### 3. 策略参数

**JSON格式输入:**
```json
{
  "period": 20,
  "threshold": 0.02
}
```

- 留空使用策略默认参数
- 点击"📖 查看策略详情"查看每个策略的默认参数

## ⚙️ 回测配置标签页

### 1. 初始资金
- 默认: 200,000 元
- 建议: 根据实际情况设置

### 2. 手续费率
- 默认: 0.0001 (0.01%)
- 说明: 买卖双向各收一次
- A股常用: 0.0003 (0.03%)

### 3. 滑点
- 默认: 0.001 (0.1%)
- 说明: 模拟实际成交价格与理论价格的偏差

### 4. 复权方式
- **空**: 不复权
- **qfq**: 前复权（推荐）
- **hfq**: 后复权

### 5. 输出目录
- 默认: `./reports_gui`
- 所有回测结果将保存在此目录
- 点击"浏览..."选择其他目录

### 6. 选项
- ☑ **生成图表**: 生成回测可视化图表（推荐）
- ☑ **显示详细日志**: 在输出区显示详细过程

## 🔍 优化配置标签页

### 1. 运行模式

#### ○ 单次回测
- 使用指定参数运行一次回测
- 适合: 验证特定参数组合

#### ○ 网格搜索
- 遍历参数网格，找出最优组合
- 需要配置"网格搜索参数"
- 示例:
  ```json
  {
    "period": [5, 10, 20, 30],
    "threshold": [0.01, 0.02, 0.05]
  }
  ```
- 结果: 保存为 `grid_<策略名>.csv`

#### ○ 自动化流程 (推荐)
- 多策略 × 多股票 × 参数优化
- 自动生成 Pareto 前沿分析
- 输出 Top-N 最优配置详细报告

### 2. 并行进程数
- 默认: 4
- 建议: CPU核心数 - 1
- 范围: 1 ~ 16

### 3. Top N
- 默认: 5
- 说明: 自动化流程中重跑前N名配置
- 范围: 1 ~ 20

### 4. 最小交易次数
- 默认: 1
- 说明: 筛选 Top-N 时要求的最小交易次数
- 避免选择几乎不交易的配置

### 5. 高级选项

#### ☑ 使用 Hot-Only 模式
- 缩小参数搜索范围
- 只搜索经过验证的"热区"参数
- 优点: 速度快，结果更稳定

#### ☑ 使用基准趋势过滤
- 基于基准指数 EMA200 判断牛熊
- 只在牛市时开仓
- 配合"趋势过滤范围"使用

#### 趋势过滤范围
- **trend**: 只对趋势策略过滤（推荐）
- **all**: 对所有策略过滤
- **none**: 不过滤

### 6. 网格搜索参数

仅在"网格搜索"模式下使用：

**示例1: 优化单个参数**
```json
{
  "period": [10, 20, 30, 40, 50]
}
```

**示例2: 优化多个参数**
```json
{
  "fast_period": [5, 10, 15],
  "slow_period": [30, 40, 50],
  "threshold": [0.01, 0.02]
}
```

**留空**: 使用策略默认网格

## 🎮 操作流程

### 基础流程（单次回测）

1. **配置数据**
   - 选择数据源: akshare
   - 输入股票代码: 600519.SH
   - 设置日期范围: 2022-01-01 ~ 2024-12-31
   - 点击"预览数据"验证

2. **选择策略**
   - 选择一个策略: ema
   - 策略参数留空（使用默认）

3. **配置回测**
   - 初始资金: 200000
   - 其他参数保持默认

4. **选择模式**
   - 运行模式: 单次回测

5. **开始回测**
   - 点击"▶️ 开始回测"
   - 观察右侧日志输出
   - 等待完成

6. **查看结果**
   - 日志区显示指标摘要
   - 输出目录中查看详细报告

### 高级流程（自动化流程）

1. **配置数据**
   - 输入多个股票代码（3-10只）
   - 设置较长日期范围（2-3年）
   - 设置基准指数: 000300.SS

2. **选择策略**
   - 选择多个策略（3-8个）
   - 建议混合趋势和震荡策略

3. **配置优化**
   - 运行模式: 自动化流程
   - 并行进程数: 4-6
   - Top N: 5
   - ☑ Hot-Only 模式
   - ☑ 基准趋势过滤（可选）

4. **开始回测**
   - 点击"▶️ 开始回测"
   - 这可能需要 1-5 分钟

5. **查看结果**
   - `reports_gui/pareto_front.png` - Pareto前沿图
   - `reports_gui/opt_*.csv` - 每个策略的优化结果
   - `reports_gui/rerun_rank_*` - Top-N 详细报告

## 🎯 最佳实践

### 1. 初次使用

- 先用"单次回测"模式熟悉界面
- 只选1只股票 + 1个策略
- 日期范围不要太长（1年即可）

### 2. 参数优化

- 用"网格搜索"模式
- 先用小网格测试（2-3个值）
- 确认可行后扩大网格

### 3. 批量分析

- 用"自动化流程"模式
- 开启 Hot-Only 提升速度
- 合理设置并行进程数
- 查看 Pareto 前沿找最优解

### 4. 数据管理

- 定期清理缓存目录（超过100MB时）
- 为不同项目使用不同输出目录
- 保存常用配置到文件

### 5. 性能优化

- 减少股票数量（<10只）
- 减少策略数量（<8个）
- 增加并行进程数（CPU允许情况下）
- 启用 Hot-Only 模式

## ❓ 常见问题

### Q1: GUI 无法启动

**A**: 检查以下几点：
1. Python 版本 >= 3.9
2. 依赖已安装: `pip install -r requirements.txt`
3. 在项目根目录运行: `python backtest_gui.py`

### Q2: 数据预览失败

**A**: 可能原因：
- 股票代码格式错误（应为 600519.SH 而非 600519）
- 日期格式错误（应为 YYYY-MM-DD）
- 网络连接问题（AKShare 需要访问外网）

### Q3: 回测运行很慢

**A**: 优化方法：
- 减少股票数量
- 减少策略数量
- 缩短日期范围
- 增加并行进程数
- 启用 Hot-Only 模式

### Q4: 找不到输出结果

**A**: 检查：
1. 回测是否成功完成（查看日志）
2. 输出目录路径是否正确
3. 是否有写入权限

### Q5: 策略参数怎么填

**A**: 
1. 留空使用默认参数（推荐）
2. 点击"查看策略详情"查看可用参数
3. 使用 JSON 格式: `{"param1": value1, "param2": value2}`

### Q6: 如何停止运行中的回测

**A**: 
- 点击"⏹️ 停止"按钮
- 会弹出确认对话框
- 确认后停止（已完成部分会保存）

### Q7: 配置文件在哪

**A**: 
- 保存时可以选择任意位置
- 建议保存在项目根目录
- 参考示例: `gui_config_example.json`
