# 性能基准规范（回测平台）

## 目标
建立可重复、可比较的性能基准，支持版本回归与容量规划。

## 基线场景

### 场景 A：单策略回测
- 标的数量：100
- 数据区间：5 年日线
- 策略：趋势类（如 EMA/MACD）
- 目标：P95 完成时间、峰值内存、错误率

### 场景 B：多策略并行
- 标的数量：300
- 策略数量：10
- 并发作业：10
- 目标：总吞吐（jobs/min）、队列等待时间

### 场景 C：参数网格
- 单策略参数组合：200+
- 标的数量：50
- 目标：单位参数组合耗时、CPU 利用率

## 指标定义
- `latency_p50/p95/p99`: 作业完成延迟。
- `throughput`: 单位时间完成作业数。
- `error_rate`: 失败作业比例。
- `queue_wait_time`: 从提交到开始运行时间。
- `memory_peak_mb`: 峰值内存。
- `cpu_avg_percent`: 平均 CPU 占用。

## 数据与环境约束
- 固定数据快照：同一版本数据源和缓存状态。
- 固定硬件规格：CPU 核数、内存、磁盘类型写入报告。
- 固定软件版本：Python、依赖版本、系统信息。

## 执行流程
1. 清理非必要后台任务，记录环境信息。
2. 执行场景 A/B/C 各 3 次，取中位与离群分析。
3. 收集 API 指标（`/metrics`）与进程指标。
4. 输出报告：与上一个稳定版本做差异对比。

- 场景结果（A/B/C）：
- 回归风险：
- 结论与建议：

## 验收阈值建议（可调整）
- 单策略场景 P95 <= 30 分钟（100 标的，5 年日线）。
- 并发场景错误率 < 1%。
- 性能回归阈值：任一核心指标退化 > 15% 触发阻断评审。
