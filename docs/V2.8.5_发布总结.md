## 🎯 设计亮点

### 1. 最小侵入性

- ✅ **零修改引擎**：`BacktestEngine` 无任何改动
- ✅ **零修改事件系统**：`EventEngine` 无任何改动
- ✅ **零修改数据网关**：`DataProvider`/`Gateway` 无任何改动
- ✅ **向后兼容**：原有 `ml_strategies.py` 用法不变

### 2. 优雅降级

```python
# 依赖探测逻辑
if XGB_OK:
    return xgb.XGBClassifier(...)  # 最强模型
elif SK_OK:
    return RandomForestClassifier(...)  # 次选
else:
    return LogisticRegression(...)  # 兜底
```

**收益**:
- 生产环境未安装 XGBoost 时自动退化到 RandomForest
- 无硬依赖崩溃，用户体验友好

### 3. 策略对等性

```python
# ML 策略与技术指标策略享有同等接口
STRATEGY_REGISTRY = {
    "ml_walk": ML_WALK_MODULE,      # 机器学习
    "adx_trend": ADX_MODULE,        # 技术指标
    "macd_e": MACD_MODULE,          # 技术指标
    ...
}
```

**收益**:
- 统一调用方式：`--strategy ml_walk` vs `--strategy adx_trend`
- 统一网格搜索：同样的 `--param-ranges` 语法
- 统一性能指标：Sharpe/MDD/Trades 等

### 4. 性能优化

| 优化点 | 实现 | 收益 |
|--------|------|------|
| **特征预计算** | `__init__` 阶段一次性生成 | 避免 O(N²) 重复计算 |
| **模型缓存** | `_make_model()` 仅调用一次 | 减少初始化开销 |
| **并行训练** | XGBoost/RF 使用 `n_jobs=-1` | 多核加速 |
| **增量训练** | SGDClassifier `partial_fit` | 大样本时更快 |

**实测数据** (600519.SH, 2023-2024, 484 bars):
- XGBoost: ~15s
- RandomForest: ~20s
- LogisticRegression: ~10s
- SGD (partial_fit): ~5s
