**版本**: V2.8.3.3

## 问题描述

用户报告发现了第三个绘图问题：

### 现象

在查看 MACD 策略的图表时，发现底部多了一个子图：

```
子图7: CCI
子图8: CrossOver 0.00  ❌ 额外的空子图
```

### 用户反馈

> "MACD 的 CrossOver 指示线额外占了一个子图（最后一行'CrossOver 0.00'）"

### 问题影响

- 所有使用 CrossOver 的 MACD 策略都有这个问题
- MACDStrategy, MACDZeroCrossStrategy 都会生成额外子图
- 图表布局变成 8 个子图（应该是 7 个）

---

## 根本原因分析

### CrossOver 指标的默认行为

Backtrader 的 `CrossOver` 指标默认会绘制：
- **1**: 上穿（金叉）
- **0**: 无交叉
- **-1**: 下穿（死叉）

### 问题代码

```python
# src/strategies/macd_backtrader_strategy.py

class MACDStrategy(bt.Strategy):
    def __init__(self):
        self.macd = bt.indicators.MACD(...)
        # ❌ 问题：CrossOver 默认创建子图
        self.crossover = bt.indicators.CrossOver(self.macd.macd, self.macd.signal)
```

### 为什么会创建子图？

1. **CrossOver 是一个指标**: Backtrader 会尝试绘制所有指标
2. **没有指定 plot 参数**: 默认 `plot=True`
3. **没有指定 plotmaster**: 创建独立子图
4. **数值范围小**: 0/1/-1 的数值，看起来像 "CrossOver 0.00" 的空白线

### 实际影响

虽然 CrossOver 只是内部逻辑用（判断金叉死叉），但 Backtrader 仍然为它创建了一个子图，导致：
- 子图数量从 7 个变成 8 个
- 每个子图高度被压缩
- 底部有个"无意义"的 CrossOver 子图

---

## 解决方案

### 核心原则

**CrossOver 只用于逻辑判断，不需要显示**：
- 策略代码中用 `self.crossover > 0` 判断金叉
- 策略代码中用 `self.crossover < 0` 判断死叉
- 不需要在图表上看到 CrossOver 的值

### 修复方法

在所有 MACD 策略中，为 CrossOver 添加 `plot=False` 参数。

```python
class MACD_EnhancedStrategy(bt.Strategy):
    """
    MACD 增强版策略：
    - 进出场仍然依据 MACD / Signal 金叉/死叉
    - 增加趋势过滤（EMA200 上升才允许做多）
    - 增加冷却期、最小持仓 bars、止损/止盈
    """
    params = (
        ("fast", 12),
        ("slow", 26),
        ("signal", 9),
        ("ema_trend_period", 200),
        ("trend_filter", True),
        ("cooldown", 5),             # 平仓后的冷却期
        ("min_hold", 3),             # 最小持仓 bars
        ("stop_loss_pct", 0.05),     # 5% 止损
        ("take_profit_pct", 0.10),   # 10% 止盈
        ("printlog", False),
    )

    def __init__(self):
        self.macd = bt.indicators.MACD(...)
        # ✅ 关闭绘图，避免子图
        self.crossover = bt.indicators.CrossOver(self.macd.macd, self.macd.signal, plot=False)
        self.ema_trend = bt.indicators.ExponentialMovingAverage(
            self.data.close, 
            period=self.params.ema_trend_period, 
            plot=False  # ✅ 趋势线也不显示
        )
        self.ema_trend_slope = self.ema_trend - self.ema_trend(-1)
        ...

    def next(self):
        # 止损/止盈逻辑
        if self.position:
            if self.params.take_profit_pct > 0 and self.entry_price:
                if close >= self.entry_price * (1.0 + self.params.take_profit_pct):
                    self.order = self.sell()  # 止盈
            if self.params.stop_loss_pct > 0 and self.entry_price:
                if close <= self.entry_price * (1.0 - self.params.stop_loss_pct):
                    self.order = self.sell()  # 止损

        # 冷却期检查
        if (i - self.last_exit_bar) < int(self.params.cooldown):
            return

        # 趋势过滤
        trend_ok = True
        if self.params.trend_filter:
            trend_ok = (self.ema_trend_slope[0] > 0)

        # 进出场逻辑（结合趋势过滤）
        if not self.position:
            if trend_ok and self.crossover > 0:
                self.order = self.buy()
        else:
            if self.crossover < 0:
                self.order = self.sell()
```

## MACD 增强策略详解

### 设计理念

**原版 MACD 的问题**:
- 在震荡区间会产生大量假信号
- 没有止损止盈机制
- 频繁交易导致手续费损失

**增强版的改进**:
1. **趋势过滤**: EMA200 向上才做多，过滤震荡区间
2. **冷却期**: 平仓后休息几根 bar，避免连续亏损
3. **最小持仓**: 避免"一买就卖"的无效交易
4. **止损止盈**: 自动风险管理

### 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `fast` | 12 | MACD 快线周期 |
| `slow` | 26 | MACD 慢线周期 |
| `signal` | 9 | MACD 信号线周期 |
| `ema_trend_period` | 200 | 趋势过滤均线周期 |
| `trend_filter` | True | 是否启用趋势过滤 |
| `cooldown` | 5 | 平仓后的冷却期（bar 数）|
| `min_hold` | 3 | 最小持仓 bar 数 |
| `stop_loss_pct` | 0.05 | 止损百分比（5%）|
| `take_profit_pct` | 0.10 | 止盈百分比（10%）|

## 常见问题

### Q1: CrossOver 还能用吗？

**A**: 完全可以！`plot=False` 只是不显示，逻辑依然正常：

```python
# 策略代码中仍然可以使用
if self.crossover > 0:  # 金叉
    self.buy()
elif self.crossover < 0:  # 死叉
    self.sell()
```

### Q2: 如何查看 CrossOver 的值？

**A**: 如果调试时需要查看，可以临时改回：

```python
# 调试时显示 CrossOver
self.crossover = bt.indicators.CrossOver(self.macd.macd, self.macd.signal, plot=True)
```

### Q3: 增强版适合什么场景？

**A**: 
- ✅ **震荡市**: 减少假信号，避免频繁止损
- ✅ **趋势市**: EMA200 过滤确保只在上升趋势做多
- ⚠️ **快速反转**: 冷却期可能错过机会
- ⚠️ **日内交易**: 参数需要调整（周期缩短）
