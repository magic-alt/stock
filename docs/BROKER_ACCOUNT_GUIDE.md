# 券商账户与接口申请指南 | Broker Account & API Onboarding

> 本文档为使用本平台进行实盘联调或仿真验证时所需的券商账户开通、SDK 申请、合规要求与
> 联系流程的详细说明。所有信息均基于券商公开资料整理，**具体条款以券商最新
> 公告为准**。

---

## 0. 总览：选择哪一种网关？

| 适用场景 | 推荐网关 | 主要门槛 | 数据/交易延迟 | 适合品种 |
|---------|---------|---------|--------------|---------|
| 个人量化、几万-几百万资金 | **xtquant (QMT)** | 普通融资融券或两融账户 + QMT 申请 | 100-500ms | A 股、ETF、可转债 |
| 私募/机构、低延迟需求 | **XTP** (中泰证券) | 机构席位或合作私募 | 1-10ms | 股票、期权 |
| 持牌机构、自营/资管 | **Hundsun UFT** | 机构柜台席位 | 1-5ms | 股票、期货、期权、债券 |
| 海外测试/教学 | `easytrader` / 模拟 | 零门槛 | N/A | A 股（模拟） |
| 期货 / 期权 | **CTP** (规划中) | 期货账户 + simnow 测试 | 1-10ms | 商品期货、股指期货、期权 |

## 1. QMT / MiniQMT (xtquant) — 个人量化首选

### 1.1 适用对象
- **个人投资者**（普通账户即可，无最低资金门槛）
- 自有策略小规模实盘验证、低频/中频交易
- **不支持**：T0、做市、撮合自营

### 1.2 开户与申请流程

#### Step 1：开立支持 QMT 的证券账户
推荐对接券商（按 QMT 开放友好度排序）：

| 券商 | QMT 类型 | 申请门槛 | 备注 |
|------|---------|---------|------|
| **国金证券** | 国金 QMT | 资金量无硬门槛，普通账户即可 | 个人量化最常用，文档完善 |
| **海通证券** | 海通 e 海通财 QMT | 普通账户 | 客户端较稳定 |
| **中信建投** | 蜻蜓点金 QMT | 普通账户 | 提供模拟环境 |
| **方正证券** | 方正小方 QMT | 普通账户 | 支持算法交易 |
| **华泰证券** | 涨乐 QMT | 通常需要 50 万资产 | 接入门槛略高，但稳定 |

**开户方式**：
1. 下载券商手机 App（如「国金佣金宝」）按提示完成在线开户（身份证 + 银行卡，约 15 分钟）
2. 在 App 内提交《QMT/算法交易服务申请》：
   - 客服路径：客户服务 → 业务办理 → 程序化交易 / 算法交易开通
   - 部分券商需要电话回访（约 1-2 个工作日）
3. **风险测评**：必须达到 **C4（积极型）** 或以上等级。如不足，可在 App 内重新答题。
4. **签署协议**：
   - 《程序化交易客户告知书》
   - 《量化交易委托协议》
   - 《证券交易接口使用协议》
5. 等待审核：1-3 个工作日。

#### Step 2：申请 QMT 客户端
- 审核通过后，券商会在 App 内推送下载链接，或在官网「下载中心 → 量化交易」获取
- 安装路径示例：`C:\国金QMT交易端\` 或 `C:\海通QMT\`
- 首次启动需要用证券账户登录，并在客户端「设置 → 接口设置」中**勾选启用 Python API**

#### Step 3：交易所认证（重要）
- 部分券商在你**首次使用 Python API 下单前**需要单独签署《**沪深交易所 API 接入承诺函**》。
- 若提交报单时收到 `-7 没有权限` 或类似错误，请联系券商客户经理处理（通常 1 天内开通）。

### 1.3 费用与限制

| 项目 | 说明 |
|------|------|
| QMT 客户端费用 | **免费**（券商不收取额外费用） |
| 佣金 | 沿用账户原佣金（通常万 2.5 - 万 3，可与客户经理谈低至万 1） |
| 印花税/过户费 | 国家规定，与手动交易一致 |
| 报单频率 | 通常 ≤3 笔/秒，单账户日内 ≤ 数千笔；超频会被柜台限制 |
| 持仓限制 | 单股不超过流通股 5% (法规) |
| API 调用限频 | xtquant 内部限频，可通过 `GatewayConfig.max_orders_per_second` 控制 |


## 2. XTP (中泰证券) — 私募与机构低延迟

### 2.1 适用对象
- **私募基金、机构席位**（持牌或在备案中）
- 中泰证券PB业务客户
- 个人也可以申请 **XTP Vega** 模拟环境（免费、无门槛）做开发测试

### 2.2 申请流程

#### 模拟环境（推荐先用此环境完成集成）
1. 访问 [XTP 官网](https://xtp.zts.com.cn/)（中泰证券极速交易系统）
2. 注册账号 → 提交「**XTP Vega 模拟仿真**」申请
3. 通常 1 个工作日内获得：
   - 模拟账号 / 密码
   - 测试服务器地址（如 `tcp://120.27.164.138:6001`）
   - SDK 下载链接（包含 Python `.pyd` / `.so`）
4. **完全免费**，仅限白天交易时段使用。

#### 生产环境
1. **必须通过中泰证券 PB 业务**开户：
   - 联系电话：021-20315678（中泰证券 PB 业务部）
   - 邮箱：pb_service@zts.com.cn
2. **必备资质**（机构）：
   - 私募管理人备案证书（中基协 SAC 编号）
   - 已发行至少 1 只产品（首发不强制，但风控加分）
   - 营业执照、产品备案文件
3. **个人**通常无法直接申请生产环境 XTP（除非走「私享版」高净值通道，门槛 1000 万+）。
4. **签署文件**：
   - 《XTP 系统接入服务协议》
   - 《极速交易风险揭示书》
   - 《数据使用与保密协议》
5. 接入流程：环境申请 → 联调测试 → 验收 → 生产开通（通常 2-4 周）。

### 2.3 费用结构（参考，2025-2026 年）

| 项目 | 模拟 | 生产 |
|------|------|------|
| 系统接入费 | 免费 | 一次性 5,000 - 20,000 元 |
| 月度服务费 | 免费 | 1,000 - 5,000 元/月（按席位） |
| 数据费 | 含 L1 行情 | L1 免费 / L2 需另购（约 8,000 元/年/账户） |
| 佣金 | N/A | 单独议定，机构可低至万 1 - 万 1.5 |

### 2.4 数据接入差异
- **L1 行情**：tick 级（约 3 秒 1 笔，沪深）
- **L2 行情**：包含 10 档买卖盘、逐笔成交、逐笔委托，机构使用，**必须单独申请**（向上交所/深交所购买，私募一年约 8-12 万元；通过 XTP 平台分摊则成本更低）

### 2.5 常见拒绝原因
- 个人客户：无机构资质 → 仅能使用模拟
- 测试期超频报单（>50 笔/秒）→ 风控临时锁定
- 未签署 L2 数据协议而尝试申请 L2 → 直接返回权限错误

## 3. Hundsun UFT (恒生 UFT) — 机构持牌柜台

### 3.1 适用对象
- **公募基金、券商资管、自营、银行理财子**
- 已部署恒生 O45 / UFT 柜台的机构
- 个人**无法**单独申请 UFT；必须通过持牌机构

### 3.2 申请流程
> UFT 不是「开账户」概念，而是「柜台席位」概念。

1. **机构与恒生电子签署柜台合作协议**（一次性）
   - 联系恒生电子销售：021-28291888 或 sales@hundsun.com
   - 报备机构名称、业务类型、预计连接数
2. **机构内部开通席位**：
   - 由 IT 部门或对接的恒生工程师在 UFT 柜台分配一个柜员 / 通道
   - 取得：`td_front` (交易前置 IP:端口) / `md_front` (行情前置) / 账号 / 密码 / 加密狗（部分版本）
3. **SDK 申请**：恒生提供 `ufx_*.pyd` / `ufx_*.so`，需要与你部署的 UFT 版本严格匹配（V2.0/V2.5/V3.0 接口不互通）
4. **联调测试**：在机构内部测试环境跑通 connect → query_account → 模拟下单 → 撤单 → 查询的全链路
5. **风控审批**：机构风控部门审批接入策略与最大持仓后，开通生产

### 3.3 接入说明
- UFT 柜台接入需要由支持该柜台的环境统一开通。
- 个人开发者通常无法单独申请柜台席位。
- SDK 与柜台版本强绑定，具体申请条件以券商或柜台服务方最新要求为准。

### 3.4 注意事项
- UFT 接口在交易时段外**通常不可连接**，与 XTP 不同
- 部分版本的 UFT 在 Python 3.10+ 下需要使用恒生提供的兼容包；若 SDK 加载失败，请确认 SDK 编译时使用的 Python 版本
- 严禁在生产 UFT 上跑未经过测试的策略；机构内部风控会立刻拒单并报告

## 5. 数据源（akshare / tushare / yfinance）

| 数据源 | 是否需要账号 | 访问方式 | 限频 |
|--------|------------|------|------|
| AKShare | **不需要** | 公开接口 | 公网爬取，节制使用 |
| TuShare | **需要 token** | 注册后按平台积分规则开通 | 每分钟 100-500 次 |
| YFinance | **不需要** | 公开接口 | Yahoo 限频，海外网络可能不稳 |
| Qlib (Microsoft) | **不需要**（需本地数据集） | 本地数据集 | 离线计算 |

### TuShare 注册
1. 访问 [https://tushare.pro/register](https://tushare.pro/register)
2. 完成手机/邮箱注册 → 个人中心获取 token
3. 配置：
   ```bash
   export TUSHARE_TOKEN=your_token_here
   ```
   或在 `config.yaml` 中配置：
   ```yaml
   data:
     tushare:
       token: ${TUSHARE_TOKEN}
   ```
4. 部分高频/L2 数据需要满足平台积分或权限要求，具体规则以 TuShare 最新说明为准。

## 6. 合规清单（持牌机构上线前必看）

### 6.1 必备文件
- [ ] 私募管理人备案证书 / 公募牌照 / 资管牌照
- [ ] 营业执照、机构代码
- [ ] 法人代表身份证明
- [ ] 投资经理 / 量化交易员从业资格（基金从业资格证书）
- [ ] 已签署的《程序化交易报备表》（向交易所报备）

### 6.2 交易所程序化交易报备（2024 起强制）
- **沪市**：通过券商向上交所提交《程序化交易报告表》，包含策略类型、最大委托速率、撤单率、风控措施
- **深市**：与上交所类似
- **报备时点**：上线前 5 个交易日提交
- **变更报备**：策略/参数重大变更需在 3 个交易日内重新提交

### 6.3 风控基线（建议）
- 最大单笔报单金额：≤ 净资产 5%
- 单股持仓：≤ 流通股 5%（法规）/ 公司风控规则
- 日内撤单率：< 90%（高于此值可能触发监管警示）
- 单账户日内报单量：< 20,000 笔（参考线）
- 自成交（自家两边）：**禁止**，平台会自动拦截

### 6.4 数据隐私与保密
- 客户账号 / 密码 / Token 应使用 **环境变量** 或 **密钥管理服务**，**严禁**写入代码或 Git
- 平台支持 `SecurityManager` / `Vault` 抽象（见 `src/core/security.py`）
- 生产部署建议启用 TLS + 互信证书 + RBAC

如果**当前无法获得任何 SDK / 账号**，平台依然可以完成大部分开发与测试：

## 8. 升级路径建议

1. **第 1-2 周**：用 AKShare + Backtrader/Zipline 完成离线回测策略池构建
2. **第 3-4 周**：申请 QMT 个人账户，在 mini-QMT 中部署 1-2 个低频策略验证
3. **第 5-8 周**：申请 XTP Vega 模拟，把策略移植到 Python API（同样接口）做高频 / 算法测试
4. **生产上线**：
   - 个人 / 小型私募：留在 QMT，规模做大后转 XTP 生产
   - 持牌机构：通过 UFT 接入恒生柜台，由机构 IT 部门统一接口
