## 🔧 技术细节

### 插件协议设计

**FeePlugin 协议**:
```python
class FeePlugin(Protocol):
    def register(self, broker: bt.BrokerBase) -> None:
        """注册费用规则到 Broker"""
```

**SizerPlugin 协议**:
```python
class SizerPlugin(Protocol):
    def get(self) -> bt.Sizer:
        """返回 Sizer 类（不是实例）"""
```

### 插件注册机制

使用装饰器模式自动注册：
```python
@register_fee("cn_stock")
class CNStockFee(FeePlugin):
    def __init__(self, commission_rate: float = 0.0001, ...):
        ...
    def register(self, broker):
        # 创建并注册内部 CommInfoBase 类
        ...
```

### 向后兼容策略

- **默认行为不变**: 不传参数时自动加载 `cn_stock` + `cn_lot100`
- **参数化配置**: 通过 `params` 字典传入插件配置（前缀 `_`）
- **零破坏性**: 所有现有回测代码无需修改

## 🎯 架构改进

### Before (V2.6.0)
```
BacktestEngine
  └── _run_module
        ├── class CNStockCommission(bt.CommInfoBase): ...  # 107 lines
        └── class StockLotSizer(bt.Sizer): ...
```
**问题**:
- ❌ 规则硬编码在 Engine 中
- ❌ 无法扩展到其他市场
- ❌ 107 行代码嵌套在方法内

### After (V2.7.0)
```
src/bt_plugins/
  ├── base.py (注册表 + 协议)
  └── fees_cn.py (@register_fee + @register_sizer)
       
BacktestEngine
  └── _run_module
        ├── fee = load_fee("cn_stock", ...)  # 2 lines
        └── sizer = load_sizer("cn_lot100", ...)  # 2 lines
```
**优势**:
- ✅ 规则外置，职责清晰
- ✅ 易于扩展（添加 US/Crypto 插件）
- ✅ Engine 代码更简洁（净减少 82 行）
- ✅ 插件可独立测试

## 📚 使用示例

### 默认模式（A股规则）
```python
engine = BacktestEngine()
metrics = engine.run_strategy(
    "ema", 
    ["600519.SH"], 
    "2024-01-01", 
    "2024-12-31",
)
# 自动应用:
#   - cn_stock fee plugin (佣金0.01% + 印花税0.05%)
#   - cn_lot100 sizer (100股整数倍)
```

### 自定义插件参数
```python
metrics = engine.run_strategy(
    "ema",
    ["600519.SH"],
    "2024-01-01",
    "2024-12-31",
    params={
        "period": 20,
        # 插件配置（前缀 _）
        "_fee_plugin": "cn_stock",
        "_commission": 0.00015,      # 万1.5
        "_min_commission": 5.0,       # 不免五
        "_sizer_plugin": "cn_lot100",
        "_lot_size": 100,
    }
)
```
