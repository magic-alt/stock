# A股监控与回测系统 V2.0 - 使用指南

欢迎使用全新模块化的A股监控与回测系统！本指南将帮助你快速上手并充分利用系统的各项功能。

## 🚀 快速入门

### 第一步：安装依赖

```bash
cd d:\Project\data
pip install -r requirements.txt
```

### 第二步：测试连接

```bash
python test_connection.py
```

看到三个✓表示数据源正常。

### 第三步：运行主程序

```bash
python main.py
```

## 📖 功能详解

### 1. 实时监控

#### 1.1 选择监控方案

运行主程序后，选择 `1 - 实时监控`，系统提供8种方案：

**方案1：默认方案** ⭐推荐新手
- 精选AI、芯片、黄金、蓝筹各类别龙头
- 股票数量适中（约9只）
- 涵盖多个热门主题

**方案2：AI芯片主题**
- 聚焦AI和半导体行业
- 包含：中芯国际、宁德时代、歌尔股份等

**方案3：黄金主题**
- 专注黄金矿业股
- 包含：山东黄金、紫金矿业、中金黄金等
- 适合避险或看好黄金的用户

**方案4：新能源汽车**
- 电驱、锂电池概念
- 包含：卧龙电驱、赣锋锂业、比亚迪等
- 适合看好新能源的用户

**方案5：优质蓝筹**
- 传统白马股
- 包含：贵州茅台、中国平安、招商银行等
- 适合稳健投资者

**方案6：科技综合**
- 所有科技类股票
- 股票数量较多
- 适合全面了解科技板块

**方案7：全部股票**
- 监控所有分组的股票
- 数量最多，刷新较慢
- 适合做全市场扫描

**方案8：自定义**
- 手动输入股票代码
- 用逗号分隔，如：600519,000858,601318
- 适合有明确目标的用户

#### 2.2 选择策略

系统提供5种内置策略：

**策略1：双均线交叉 (MA5/MA20)** ⭐推荐新手
```
原理：短期均线与长期均线交叉
买入：MA5上穿MA20（金叉）
卖出：MA5下穿MA20（死叉）
适用：趋势明显的市场
优点：简单易懂，经典策略
缺点：震荡市容易被打脸
```

**策略2：三均线策略 (MA5/MA10/MA20)**
```
原理：三条均线的多空排列
买入：MA5>MA10>MA20（多头排列）
卖出：MA5<MA10<MA20（空头排列）
适用：强趋势市场
优点：信号更可靠
缺点：反应较慢
```

**策略3：RSI超买超卖**
```
原理：相对强弱指标的极值
买入：RSI<30（超卖）
卖出：RSI>70（超买）
适用：震荡市场
优点：逆向思维，抄底逃顶
缺点：趋势市场效果差
```

**策略4：MACD信号**
```
原理：MACD与Signal线的交叉
买入：MACD上穿Signal线
卖出：MACD下穿Signal线
适用：中期趋势跟踪
优点：平滑性好
缺点：有滞后性
```

**策略5：MACD零轴**
```
原理：MACD穿越零轴
买入：MACD上穿零轴
卖出：MACD下穿零轴
适用：判断大趋势
优点：信号明确
缺点：交易次数少
```

#### 2.3 选择回测周期

**选项1：短期 (30天)**
- 适合测试短线策略
- 数据量小，快速验证

**选项2：中期 (90天)**
- 较为平衡的选择
- 包含多个波段

**选项3：长期 (365天)** ⭐推荐
- 完整的年度周期
- 包含牛熊转换
- 结果更有参考价值

**选项4：自定义**
- 输入开始和结束日期
- 格式：YYYYMMDD
- 示例：20240101 到 20251009

#### 2.4 回测结果解读

```
================================================================================
                                    回测报告                                     
================================================================================
股票: 卧龙电驱(600580)
策略: MA交叉策略(5/20)
--------------------------------------------------------------------------------
初始资金: 100,000.00 元
最终资金: 125,500.00 元
总收益率: +25.50%
买入持有收益率: +18.30%
相对收益: +7.20%
--------------------------------------------------------------------------------
最大回撤: -12.50%
夏普比率: 1.85
交易次数: 12
胜率: 66.67%
平均盈利: 2,125.00 元 (8.50%)
```

**关键指标说明**：

1. **总收益率**
   - 策略的最终收益
   - 正数表示盈利，负数表示亏损

2. **买入持有收益率**
   - 如果在第一天买入并持有到最后的收益
   - 用于对比策略是否跑赢市场

3. **相对收益**
   - 策略收益 - 买入持有收益
   - 正数说明策略有效
   - 负数说明策略不如持有

4. **最大回撤**
   - 资金从最高点到最低点的最大跌幅
   - 衡量风险的重要指标
   - 一般期望小于-20%

5. **夏普比率**
   - 风险调整后的收益指标
   - >1 较好
   - >2 优秀
   - >3 卓越

6. **交易次数**
   - 策略的交易频率
   - 过高说明过度交易
   - 过低可能错过机会

7. **胜率**
   - 盈利交易占总交易的比例
   - >50% 说明策略整体有效
   - 注意：高胜率不一定高收益

8. **平均盈利**
   - 每笔交易的平均盈亏
   - 包含金额和百分比
   - 正数表示总体盈利

## 💡 使用技巧

### 技巧1：主题轮动

根据市场热点切换监控方案：

```
科技股活跃 → AI芯片主题
黄金上涨   → 黄金主题  
政策利好   → 新能源汽车
市场下跌   → 优质蓝筹（避险）
```

### 技巧2：策略组合

对同一股票测试多个策略，选择最优：

```
1. 测试双均线策略
2. 测试RSI策略
3. 测试MACD策略
4. 对比收益和回撤
5. 选择夏普比率最高的
```

### 技巧3：参数优化

修改策略参数寻找最优组合：

```python
# 测试不同的均线周期
测试1: MA(5, 20)
测试2: MA(5, 30)
测试3: MA(10, 20)
测试4: MA(10, 30)

# 选择收益最高且回撤可控的组合
```

### 技巧4：多周期验证

用不同周期测试策略稳定性：

```
短期(30天)  → 看短期表现
中期(90天)  → 看波段适应性
长期(365天) → 看长期稳定性

三个周期都表现好 → 策略可靠
```

### 技巧5：分批建仓

根据监控信号分批操作：

```
MA5接近MA20 → 准备买入
MA5上穿MA20 → 买入1/3
继续上涨    → 买入1/3
确认趋势    → 买入剩余1/3
```

## ⚠️ 注意事项

### 1. 数据延迟

- 免费数据源有15分钟延迟
- 不适合做高频交易
- 适合中长线参考

### 2. 回测陷阱

- **未来函数**：确保不使用未来数据
- **过拟合**：参数不要过度优化
- **交易成本**：系统已包含，但实际可能更高
- **滑点**：实际成交价格可能偏离
- **历史≠未来**：过去表现不代表未来

### 3. 风险控制

- **止损**：设置最大亏损退出点
- **止盈**：设置目标盈利出局点
- **仓位**：单只股票不超过30%
- **分散**：不要全仓一只股票

### 4. 交易建议

- **不追高**：避免在涨停板买入
- **不杀跌**：避免在跌停板卖出
- **耐心等待**：等待好的买入时机
- **纪律执行**：严格按策略执行

### 5. 法律免责

- 本工具仅供学习研究
- 不构成投资建议
- 投资有风险，入市需谨慎
- 盈亏自负

## 🐛 故障排除

### 问题1：无法获取数据

**现象**：显示"数据获取失败"

**解决**：
```bash
1. 检查网络连接
2. 运行测试: python test_connection.py
3. 等待几秒后重试
4. 检查防火墙设置
```

### 问题2：技术指标显示N/A

**原因**：历史数据不足

**解决**：
- 该股票可能是新股
- 或近期停牌时间较长
- 选择上市时间较长的股票

### 问题3：程序运行缓慢

**原因**：监控股票过多或刷新太频繁

**解决**：
```python
# 1. 减少监控股票数量
# 2. 关闭技术指标显示
# 3. 增加刷新间隔
REFRESH_INTERVAL = 60  # 改为60秒

# 4. 使用快速模式（只看价格）
python quick_monitor.py
```

### 问题4：导入错误

**现象**：ModuleNotFoundError

**解决**：
```bash
# 确保在正确目录
cd d:\Project\data

# 重新安装依赖
pip install -r requirements.txt

# 检查Python版本（需要3.7+）
python --version
```

### 问题5：回测结果不合理

**检查**：
1. 股票代码是否正确
2. 时间周期是否合理
3. 该股票在此期间是否有重大事件
4. 检查是否停牌或退市

## 🎓 常见问题FAQ

**Q1: 为什么要选V2.0？**
A: V2.0采用模块化架构，代码结构清晰，易于扩展，支持更多功能。

**Q2: 可以同时监控多少只股票？**
A: 理论上无限制，但建议不超过20只，否则刷新会很慢。

**Q3: 可以监控港股或美股吗？**
A: 当前版本仅支持A股，未来可能扩展其他市场。

**Q4: 回测结果可靠吗？**
A: 回测考虑了交易成本，但无法完全模拟实盘，仅供参考。

**Q5: 如何选择合适的策略？**
A: 不同市场适合不同策略，建议多策略对比，选择稳定性好的。

**Q6: 可以用于实盘交易吗？**
A: 本系统是学习工具，不建议直接用于实盘，需要更多验证。

**Q7: 如何获得实时数据？**
A: 免费数据源有延迟，需要实时数据请使用付费接口。

**Q8: 可以添加自己的指标吗？**
A: 可以，在`src/indicators/technical.py`中添加计算函数。

**Q9: 支持自动交易吗？**
A: 当前版本不支持，未来可能添加接口。

**Q10: 如何贡献代码？**
A: 欢迎提交改进建议和代码，保持模块化结构。
