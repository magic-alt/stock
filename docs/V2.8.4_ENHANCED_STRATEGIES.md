# V2.8.4 Enhanced Strategies Guide

## 概述

V2.8.4 新增两个增强型策略，专注于提高盈利质量、降低噪音交易：

### 核心设计理念

1. **金叉≠趋势**：通过 EMA200 斜率 + ROC100 双过滤器，避免震荡期交易
2. **不追、等回落**：强势 → 回落 → 再走强的入场模式，提高期望值
3. **以 ATR 为"货币"**：止损/追踪/止盈均采用 ATR 作为单位，保持一致性
4. **冷却/最小持有**：防止过度交易，提高稳定性
5. **保留 Backtrader 原生资金管理**：不修改仓位管理逻辑

---

## 1. Bollinger_EnhancedStrategy (boll_e)

### 策略描述

基于布林带的反弹策略，增加 ATR 动态止损、多级分批止盈、回落出场机制。

### 入场条件

1. **反弹信号**：价格从下轨反弹
   - `close[-1] < lower[-1]` AND `close[0] > lower[0]`
2. **趋势过滤**（可选）：
   - 中轨斜率 > 0（`trend_filter=True`）
3. **预热期**：
   - 前 `warmup_bars` 根K线不交易
4. **冷却期**：
   - 距离上次出场至少 `cooldown` 根K线

### 出场机制

| 机制 | 条件 | 动作 |
|------|------|------|
| **ATR 止损** | `close < entry - atr_mult_sl * ATR` | 全部平仓 |
| **第一止盈** | `close >= entry * (1 + tp1_pct)` | 平仓 50% |
| **第二止盈** | `close >= entry * (1 + tp2_pct)` | 平仓剩余全部 |
| **回落出场** | `close < highest * (1 - trail_drop_pct)` | 平仓全部 |
| **最小持有期** | `持有时间 < min_hold` | 不触发回落出场 |

### 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `period` | 20 | 布林带周期 |
| `devfactor` | 2.0 | 标准差倍数 |
| `atr_period` | 14 | ATR 周期 |
| `atr_mult_sl` | 2.5 | ATR 止损倍数 |
| `tp1_pct` | 0.03 | 第一止盈：+3% |
| `tp1_frac` | 0.5 | TP1 平仓比例：50% |
| `tp2_pct` | 0.06 | 第二止盈：+6% |
| `tp2_frac` | 1.0 | TP2 平仓比例：100% |
| `trail_drop_pct` | 0.04 | 回落阈值：-4% |
| `min_hold` | 3 | 最小持有 3 根K线 |
| `cooldown` | 5 | 冷却期 5 根K线 |
| `warmup_bars` | 30 | 预热期 30 根K线 |
| `trend_filter` | True | 是否启用趋势过滤 |

### 使用示例

```bash

  --strategy boll_e \
  --symbols 600519.SH \
  --start 2023-01-01 --end 2024-12-31 \
```

## 2. MACD_RegimePullback (macd_r)

### 策略描述

基于 MACD 的趋势回落策略，通过 EMA200 + ROC100 双过滤器识别趋势，等待回落至 EMA20 附近再入场。

### 入场条件

1. **趋势过滤**（双条件）：
   - EMA200 斜率 > 0（长期上涨趋势）
   - ROC100 > 0（中期动能正向）
2. **金叉信号**：
   - MACD > Signal
3. **回落检测**：
   - 金叉后 `max_lag` 根K线内
   - 价格回落至 `pullback_line = EMA20 - pullback_k * ATR`
   - 最低价触及回落线：`low <= pullback_line`
4. **反弹确认**：
   - 收盘价重回 EMA20 上方：`close > EMA20`

### 出场机制

| 机制 | 条件 | 动作 |
|------|------|------|
| **ATR 初始止损** | `close < entry - atr_sl_mult * ATR` | 全部平仓 |
| **ATR 追踪止损** | `close < highest - atr_trail_mult * ATR` | 全部平仓 |
| **第一止盈 (R)** | `close >= entry + tp1_R * R` | 平仓 50% |
| **第二止盈 (R)** | `close >= entry + tp2_R * R` | 平仓剩余全部 |
| **最小持有期** | `持有时间 < min_hold` | 不触发追踪止损 |

**注**：R = `atr_sl_mult * ATR`（初始风险单位）

### 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `fast` | 12 | MACD 快线周期 |
| `slow` | 26 | MACD 慢线周期 |
| `signal` | 9 | MACD 信号线周期 |
| `ema_trend_period` | 200 | 趋势过滤 EMA 周期 |
| `roc_period` | 100 | 趋势过滤 ROC 周期 |
| `trend_filter` | True | 是否启用趋势过滤 |
| `ema_entry_period` | 20 | 入场参考 EMA 周期 |
| `pullback_k` | 0.5 | 回落线系数（k*ATR） |
| `max_lag` | 5 | 金叉后最大等待周期 |
| `atr_period` | 14 | ATR 周期 |
| `atr_sl_mult` | 2.5 | ATR 初始止损倍数 |
| `atr_trail_mult` | 2.0 | ATR 追踪止损倍数 |
| `min_hold` | 3 | 最小持有 3 根K线 |
| `cooldown` | 5 | 冷却期 5 根K线 |
| `tp1_R` | 1.0 | TP1 目标：1R |
| `tp1_frac` | 0.5 | TP1 平仓比例：50% |
| `tp2_R` | 2.0 | TP2 目标：2R |

## 3. 图表优化

### indicator_preset 参数

V2.8.4 新增 `indicator_preset` 参数，支持两种模式：

| 模式 | 子图 | 说明 |
|------|------|------|
| `"clean"` | 主图 + 成交量 + MACD | 默认模式，清爽简洁 |
| `"full"` | 主图 + 成交量 + MACD + ADX + RSI + Stoch + CCI | 完整模式，所有指标 |

### 使用方法

```python
# 在 plotting.py 中调用
plot_backtest_with_indicators(
    cerebro, 
    strategy_name='boll_e',
    indicator_preset='clean'  # 或 'full'
)
```

### 其他优化

- **voloverlay**: 修复为 `True`，成交量均线叠加在成交量面板
- **更快渲染**：clean 模式减少子图数量，加快绘图速度
- **更小文件**：clean 模式图表文件约 300-400KB，full 模式约 420-500KB

## 4. 性能特点

### 交易频率

两个策略均设计为**低频高质量**：

- **boll_e**：反弹 + 趋势过滤 → 预计每年 5-15 次交易
- **macd_r**：双趋势 + 回落 → 预计每年 3-10 次交易

### 风险控制

| 策略 | 止损机制 | 止盈机制 | 特点 |
|------|----------|----------|------|
| boll_e | ATR 固定止损 + 回落追踪 | 分批止盈（3%/6%） | 灵活出场 |
| macd_r | ATR 初始 + ATR 追踪 | R 单位止盈（1R/2R） | 风险统一 |

### 适用场景

| 策略 | 适用市场 | 不适用市场 |
|------|----------|------------|
| boll_e | 震荡上涨、箱体突破 | 单边下跌、无波动 |
| macd_r | 中长期趋势、牛市 | 震荡市、熊市 |

## 5. 参数调优建议

### 增加交易频率

如果测试结果显示 0 交易或交易太少：

1. **关闭趋势过滤**：
   ```bash
   --params '{"trend_filter": false}'
   ```

2. **缩短预热期**：
   ```bash
   --params '{"warmup_bars": 20}'
   ```

3. **放宽回落条件** (macd_r)：
   ```bash
   --params '{"pullback_k": 0.3, "max_lag": 10}'
   ```

### 提高风险收益比

如果希望更激进的止盈：

1. **调整止盈比例** (boll_e)：
   ```bash
   --params '{"tp1_pct": 0.05, "tp2_pct": 0.12}'
   ```

2. **调整 R 倍数** (macd_r)：
   ```bash
   --params '{"tp1_R": 1.5, "tp2_R": 3.0}'
   ```

### 降低风险

如果回撤过大：

1. **收紧止损**：
   ```bash
   --params '{"atr_mult_sl": 2.0}'  # boll_e
   --params '{"atr_sl_mult": 2.0}'  # macd_r
   ```

2. **提前止盈**：
   ```bash
   --params '{"tp1_pct": 0.02, "tp1_frac": 0.7}'  # boll_e
   --params '{"tp1_R": 0.8, "tp1_frac": 0.6}'     # macd_r
   ```

## 7. 常见问题

### Q1: 为什么测试结果显示 0 交易？

**A**: 这是正常现象，说明过滤器正在工作。可能原因：

1. **熊市环境**：2024 年市场整体下跌，EMA200 + ROC100 双过滤拒绝交易
2. **过滤太严格**：`trend_filter=True` + 回落条件 → 很少触发
3. **测试周期太短**：建议测试 2-3 年，覆盖牛熊周期

**解决方案**：
- 延长测试周期至 2023-2024
- 关闭趋势过滤：`--params '{"trend_filter": false}'`
- 选择牛市标的或周期

### Q2: 如何调整仓位管理？

**A**: 这些策略保留了 Backtrader 原生仓位管理，需在框架层面配置：

```bash
# 在 unified_backtest_framework.py 中设置
cerebro.broker.set_cash(200000)  # 初始资金
cerebro.broker.set_commission(commission=0.001)  # 手续费
# 仓位由 Backtrader 的 Sizer 控制
```

### Q3: 参数太多，如何选择？

**A**: 使用网格搜索自动优化：

```bash
python unified_backtest_framework.py grid \
  --strategy boll_e \
  --symbols 600519.SH \
  --start 2023-01-01 --end 2024-12-31
```

系统会自动测试 `grid_defaults` 中定义的参数组合，输出最佳结果。

### Q4: 图表太复杂，如何简化？

**A**: 使用 clean 模式（默认已启用）：

```python
# 在代码中
indicator_preset='clean'  # 只显示主图 + 成交量 + MACD

# 或切换为完整模式
indicator_preset='full'   # 显示所有指标
```

### Q5: 如何与原策略对比？

**A**: 同时运行两个版本：

```bash
# 原始策略
python unified_backtest_framework.py run --strategy macd --symbols 600519.SH ...

# 增强策略
python unified_backtest_framework.py run --strategy macd_r --symbols 600519.SH ...

# 对比指标：
# - cum_return (累计收益率)
# - max_drawdown (最大回撤)
# - sharpe_ratio (夏普比率)
# - trades (交易次数)
# - win_rate (胜率)
```
