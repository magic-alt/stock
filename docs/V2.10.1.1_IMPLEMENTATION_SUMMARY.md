## 核心改进

### 1. 优化数据库架构 ⭐⭐⭐⭐⭐

**问题**: V2.10.1使用统一表（stock_daily, index_daily）存储所有数据，需要symbol过滤

**解决方案**: 每个股票/指数创建独立表
- `stock_600519_SH` - 贵州茅台数据
- `stock_000001_SZ` - 平安银行数据
- `index_000300_SH` - 沪深300指数
- `index_GSPC` - 标普500指数

**优势**:
- ✅ 查询性能提升（无需symbol过滤）
- ✅ 支持多样化标的代码（国际指数）
- ✅ 数据管理更直观
- ✅ 导入导出更简单

### 2. 国际指数支持 🌍

支持的指数类型：
- **A股**: 000300.SH（沪深300）, 000001.SH（上证指数）, 399001.SZ（深证成指）
- **港股**: ^HSI（恒生指数）
- **美股**: ^GSPC（标普500）, ^DJI（道琼斯）, ^IXIC（纳斯达克）

表名转换示例：
```
600519.SH  → stock_600519_SH
^GSPC      → index_GSPC
000300.SH  → index_000300_SH
```

### 3. CSV导入功能 📥

**新增功能**:
- `import_from_csv()`: 单个CSV文件导入
- `batch_import_from_cache()`: 批量导入cache目录

**特性**:
- 支持中英文列名（日期/date, 开盘/open, 收盘/close等）
- 自动识别股票/指数类型
- 智能解析文件名: `ak_SYMBOL_START_END_ADJ.csv`

**实际测试**:
```

✓ 0个失败
✓ 包含多种股票代码格式
```

```sql

CREATE TABLE metadata (
    symbol TEXT NOT NULL,
    data_type TEXT NOT NULL,
    adj_type TEXT NOT NULL,
    table_name TEXT NOT NULL,      -- NEW
    start_date TEXT,                -- 改名from first_date
    end_date TEXT,                  -- 改名from last_date
    record_count INTEGER DEFAULT 0, -- NEW
    last_updated TEXT,              -- 改名from last_update

)

    adj_type TEXT
)

CREATE TABLE index_000300_SH (
    date TEXT NOT NULL PRIMARY KEY,

)
```

## 已知问题

### Windows文件锁

**问题**: 删除数据库时可能遇到文件锁
```
PermissionError: [WinError 32] 另一个程序正在使用此文件
```

**解决**:
```python
del db  # 删除对象
import gc
gc.collect()  # 强制垃圾回收
# 或重启Python
```

### CSV列名识别

**问题**: 部分CSV可能有不同的列名格式

**解决**: 系统已支持常见中英文列名映射，特殊格式需要预处理
