### 1. ✅ 策略模板抽象（StrategyTemplate Protocol）

**目标**: 提供简化的、框架无关的策略开发接口

**已实现文件**:
- `src/strategy/template.py`: StrategyTemplate 协议定义 + BacktraderAdapter
- `src/strategies/ema_template.py`: EMA 策略模板示例（完整）
- `src/strategies/macd_template.py`: MACD 策略模板示例（完整）

**核心特性**:
- ✅ 简化的生命周期方法（`on_init`, `on_start`, `on_bar`, `on_stop`）
- ✅ 框架无关设计（可适配 Backtrader 或 PaperRunner）
- ✅ 清晰的状态管理（`self.params`, `self.ctx`）
- ✅ 自动适配器（`BacktraderAdapter.to_bt_strategy()`）
- ✅ 易于测试（纯 Python，无 Backtrader 魔法）

**使用示例**:

```python
from src.strategy.template import StrategyTemplate, build_bt_strategy

# 1. 定义策略模板
class MyStrategy(StrategyTemplate):
    params = {"period": 20}
    
    def on_init(self):
        self.ctx = {}
    
    def on_bar(self, symbol: str, bar: pd.Series):
        # 访问 bar["open"], bar["close"] 等
        # 更新 self.ctx[symbol] 状态
        pass

# 2. 转换为 Backtrader 策略
MyBTStrategy = build_bt_strategy(MyStrategy, period=20)

# 3. 在 Backtest Engine 中使用
# 只需注册到 STRATEGY_REGISTRY（未来版本自动化）
```

**EMA 模板示例** (`src/strategies/ema_template.py`):
- ✅ 完整的 EMA 交叉策略实现
- ✅ 逐 bar 计算 EMA（使用 pandas.ewm）
- ✅ 交叉信号检测（价格穿越 EMA）
- ✅ 位置状态管理（0=空仓, 1=持仓）

**MACD 模板示例** (`src/strategies/macd_template.py`):
- ✅ 完整的 MACD 交叉策略实现
- ✅ 多指标计算（MACD line, Signal line, Histogram）
- ✅ 交叉检测（MACD vs Signal）
- ✅ 可选柱状图过滤器（`use_histogram_filter`）
- ✅ 前值保存（用于交叉检测）

**优势**:
- 📝 代码更简洁（无需继承 `bt.Strategy`）
- 🧪 更易测试（纯函数，可单元测试）
- 🔄 可复用（同一模板适配多个执行引擎）
- 🎯 学习曲线平缓（无需理解 Backtrader 内部机制）

### 2. ✅ CLI 参数扩展（Fee Configuration）

**目标**: 通过命令行参数配置交易费用插件

**新增参数**:

```bash
--fee-config <plugin_name>
  # Fee plugin 名称（例如：cn_stock, us_stock）
  # 如果不指定，使用默认 commission

--fee-params <json_string>
  # Fee plugin 参数（JSON 格式）
  # 例如：'{"commission_rate":0.0001,"min_commission":5.0}'
```

**支持的命令**:
- ✅ `run`: 单次回测
- ✅ `grid`: 网格搜索
- ✅ `auto`: 自动优化

**实现细节**:
- `unified_backtest_framework.py`: 添加 CLI 参数解析
- `src/backtest/engine.py`: `run_strategy()` 方法新增 `fee_plugin` 和 `fee_plugin_params` 参数
- `src/backtest/strategy_modules.py`: `add_strategy()` 过滤下划线开头的内部参数

**使用示例**:

```bash
# 1. 使用默认 A股规则（免五模式）
python unified_backtest_framework.py run \
  --strategy ema \
  --symbols 600519.SH \
  --start 2023-01-01 --end 2023-12-31 \
  --fee-config cn_stock \
  --fee-params '{"commission_rate":0.0001,"min_commission":0.0}'

# 2. 使用不免五模式（最低佣金 5 元）
python unified_backtest_framework.py run \
  --strategy ema \
  --symbols 600519.SH \
  --start 2023-01-01 --end 2023-12-31 \
  --fee-config cn_stock \
  --fee-params '{"commission_rate":0.0001,"min_commission":5.0}'

# 3. 自定义佣金率和印花税
python unified_backtest_framework.py run \
  --strategy ema \
  --symbols 600519.SH \
  --start 2023-01-01 --end 2023-12-31 \
  --fee-config cn_stock \
  --fee-params '{"commission_rate":0.0002,"stamp_tax_rate":0.0005,"min_commission":0.0}'

# 4. Grid search 使用 fee 配置
python unified_backtest_framework.py grid \
  --strategy ema \
  --symbols 600519.SH \
  --start 2023-01-01 --end 2023-12-31 \
  --fee-config cn_stock \
  --fee-params '{"commission_rate":0.0001,"min_commission":0.0}' \
  --out_csv ema_grid_results.csv
```

**内部实现流程**:

1. CLI 解析 `--fee-config` 和 `--fee-params`
2. Engine 将参数注入 `param_dict` (以下划线开头):
   ```python
   param_dict["_fee_plugin"] = "cn_stock"
   param_dict["_commission_rate"] = 0.0001
   param_dict["_min_commission"] = 0.0
   ```
3. `_run_module` 提取这些参数配置 broker:
   ```python
   fee_plugin_name = params.get("_fee_plugin", "cn_stock")
   fee = load_fee(fee_plugin_name, **fee_params)
   fee.register(cerebro.broker)
   ```
4. `add_strategy` 过滤掉下划线参数再传给策略:
   ```python
   strategy_params = {k: v for k, v in params.items() if not k.startswith('_')}
   ```

### 3. ✅ 事件驱动完善（Event-Driven Pipeline）

**目标**: 完善 Pipeline 事件处理，支持进度条、通知等功能

**新增 Handlers** (`src/pipeline/handlers.py`):

#### 3.1 ProgressBarHandler (tqdm-based)

**功能**: 实时进度条显示网格搜索进度

**特性**:
- ✅ 基于 tqdm 的实时进度条
- ✅ 显示完成进度（N/Total）
- ✅ 实时 Sharpe ratio 显示（postfix）
- ✅ 自动计时（已用时间 + 预估剩余时间）
- ✅ 优雅降级（tqdm 不可用时静默禁用）

**使用示例**:

```python
from src.backtest.engine import BacktestEngine
from src.pipeline.handlers import make_progress_bar_handler

# 创建 engine
engine = BacktestEngine()

# 注册进度条 handler
handlers = make_progress_bar_handler("Optimizing Strategy")
for etype, handler in handlers:
    engine.events.register(etype, handler)

# 启动事件系统
engine.events.start()

# 运行 grid search（进度条会自动出现）
results = engine.grid_search(
    "ema",
    {"period": [10, 20, 30, 50]},
    ["600519.SH"],
    "2023-01-01",
    "2023-12-31"
)

# 停止事件系统
engine.events.stop()
```

**输出示例**:
```
Grid Search [ema]: 100%|█████████| 4/4 [00:15<00:00, 3.75s/run] Sharpe=1.234
```

#### 3.2 TelegramNotifier

**功能**: Pipeline 完成时发送 Telegram 通知

**特性**:
- ✅ 通过 Telegram Bot API 发送消息
- ✅ Pipeline 完成通知（grid.done, auto.done）
- ✅ 可选风险告警（risk.warning）
- ✅ Markdown 格式支持
- ✅ 配置检查（优雅降级）

**配置方式**:

```bash
# 方式1: 环境变量
export TELEGRAM_BOT_TOKEN="123456:ABC-DEF..."
export TELEGRAM_CHAT_ID="987654321"

# 方式2: 代码配置
from src.pipeline.handlers import make_telegram_notifier

handlers = make_telegram_notifier(
    bot_token="123456:ABC-DEF...",
    chat_id="987654321",
    enable_risk_alerts=True
)
```

**使用示例**:

```python
from src.backtest.engine import BacktestEngine
from src.pipeline.handlers import make_telegram_notifier

engine = BacktestEngine()

# 注册 Telegram notifier
handlers = make_telegram_notifier(enable_risk_alerts=False)
for etype, handler in handlers:
    engine.events.register(etype, handler)

engine.events.start()

# 运行 auto pipeline（完成后自动发送 Telegram 消息）
engine.auto_pipeline(
    ["600519.SH", "000858.SZ"],
    "2023-01-01",
    "2023-12-31",
    strategies=["ema", "macd"],
    out_dir="./reports_auto"
)

engine.events.stop()
```

**通知消息示例**:
```
✅ Pipeline Completed
Strategy: `ema`
Stage: `grid.done`
Total runs: 120
```

#### 3.3 EmailNotifier

**功能**: Pipeline 完成时发送电子邮件通知

**特性**:
- ✅ SMTP-based email 发送
- ✅ HTML 邮件格式
- ✅ Pipeline 完成通知
- ✅ 可选风险告警
- ✅ 配置检查（优雅降级）

**配置方式**:

```bash
# 环境变量配置
export EMAIL_SMTP_HOST="smtp.gmail.com"
export EMAIL_SMTP_PORT="587"
export EMAIL_USERNAME="your@gmail.com"
export EMAIL_PASSWORD="your_app_password"
export EMAIL_FROM="your@gmail.com"
export EMAIL_TO="recipient@example.com"
```

**使用示例**:

```python
from src.backtest.engine import BacktestEngine
from src.pipeline.handlers import make_email_notifier

engine = BacktestEngine()

# 注册 Email notifier
handlers = make_email_notifier(enable_risk_alerts=False)
for etype, handler in handlers:
    engine.events.register(etype, handler)

engine.events.start()
engine.auto_pipeline(...)
engine.events.stop()
```

**邮件示例**:
```html

Strategy: ema
Stage: grid.done
Total runs: 120
```
